## O que é estrutura de dados e qual a sua importância? 🤔

As estruturas de dados definem a organização e gerenciamento de dados para realizar processos de manipulação e buscas de forma eficiente.

Torna as aplicações organizadas e facilitadas para o uso, como também resulta em um programa muito mais eficiente.

### Array 🦦

- Array é um conjunto de dados do mesmo tipo armazenados sob um único nome. Possui **fácil implementação** e **facilidade de gerenciamento**, porém é necessário criar métodos para realizar cadastros, exclusão, etc.


- Temos dois tipos de array: o **vetor** (unidimensional) e a **matriz** (multidimensional).


- Em relação à alocação de memória, existem duas formas de lidar com o array. De forma **estática**, que inicia com uma quantidade pré-definida de registros máximos, ou então **dinâmica**, com um método para aumentar sua capacidade conforme necessidade.

### Lista 🎍

- Nas listas, cada elemento possui um índice associado. Em Java, podemos implementar as listas usando a interface List, geralmente com alguma classe, como por exemplo *ArrayList* ou *LinkedList*.


- Na interface List, há alguns comandos que facilitam muito seu uso, como o comando **.size()** que retorna o número de elementos da lista.

### Fila 💫

- **Fila** ou **Queue** é uma *estrutura de dados* em que o primeiro elemento que foi inserido na fila também será o primeiro a ser removido. Não permite elementos duplicados e possui métodos específicos.

### Pilha 🐲

- **Pilha** ou **Stack** também é uma *estrutura de dados* que funciona com o princípio de “o último a entrar sairá primeiro”. Também possui métodos próprios.
